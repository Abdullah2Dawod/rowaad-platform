# إعداد بريد التجربة — Mailtrap Sandbox

منصة **رواد** تُرسل إيميلات حقيقية عند كل عملية داخل النظام (حجز جديد، تأكيد، اعتماد مستشار، توثيق البريد، تغيير كلمة السر...).

للتجربة الآمنة نستخدم **Mailtrap Sandbox** — خدمة مجانية تمسك كل الإيميلات في صندوق ويب أنيق **بدون إرسالها لصناديق البريد الحقيقية**.

## 🔧 خطوات الإعداد (5 دقائق)

### 1) أنشئ حساباً مجانياً

اذهب إلى: **https://mailtrap.io**

- اضغط **Sign Up** — يمكنك التسجيل بحساب Google أو GitHub
- لا يحتاج بطاقة ائتمانية
- الحساب المجاني يشمل: **500 إيميل شهرياً** + صندوق ويب كامل

### 2) افتح Inbox الافتراضي

بعد تسجيل الدخول:
1. من القائمة الجانبية: **Sandbox → Inboxes**
2. اضغط على **My Inbox** (يُنشأ تلقائياً)
3. اختر تبويب **SMTP Settings**
4. من القائمة المنسدلة **Integrations** اختر: **Laravel 9+**

### 3) انسخ البيانات

سيظهر لك مربع بيانات SMTP يحوي:
```env
MAIL_MAILER=smtp
MAIL_HOST=sandbox.smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=abc123def456   ← انسخ هذا
MAIL_PASSWORD=xyz789uvw012   ← وهذا
```

### 4) ضع البيانات في `.env`

افتح ملف `.env` في جذر المشروع، وابحث عن السطرين:

```env
MAIL_USERNAME=YOUR_MAILTRAP_USERNAME_HERE
MAIL_PASSWORD=YOUR_MAILTRAP_PASSWORD_HERE
```

استبدلهما بالبيانات التي نسختها من Mailtrap.

### 5) مسح الـ config cache

```bash
php artisan config:clear
```

### 6) اختبار الإرسال

```bash
php artisan mail:test
```

المتوقع:
```
Sending test email using [smtp] to: admin@rowaad.org
✓ تم إرسال الرسالة بنجاح.
  افتح صندوق Mailtrap لرؤية الرسالة: https://mailtrap.io/inboxes
```

اذهب إلى صندوقك في Mailtrap — ستجد الرسالة ✓

## 🎯 اختبار التدفقات الفعلية

### تدفق حجز الاستشارة
1. سجّل مستخدماً جديداً من `/login` → اضغط "أنشئ حساباً"
2. **إيميل توثيق** يصل Mailtrap فوراً — افتح الرابط داخل Mailtrap
3. احجز استشارة مع مستشار → أكمل الدفع
4. **إيميلان جديدان** في Mailtrap:
   - للأدمن: "حجز جديد بانتظار التأكيد"
   - للمستشار: نفس الرسالة مع تفاصيل الحجز الكاملة
5. سجّل دخول كمستشار → أكّد الحجز مع رابط Zoom
6. **إيميل جديد** للعميل: "تم تأكيد استشارتك" مع زر "الانضمام إلى الجلسة"

### تدفق اعتماد المستشار
1. اذهب إلى `/become-a-consultant` → أكمل 3 خطوات
2. **إيميل للأدمن** في Mailtrap: "طلب مستشار جديد"
3. من `/admin` اعتمد الطلب
4. **إيميل للمستشار الجديد**: كلمة المرور المُولَّدة تلقائياً + رابط `/login`

## 🛠️ إعدادات بديلة

### الوضع بدون إنترنت (Log فقط)

إذا لم يكن لديك اتصال إنترنت، غيّر في `.env`:

```env
MAIL_MAILER=log
```

كل الإيميلات ستُكتب في `storage/logs/laravel.log` بدلاً من إرسالها.

### الإنتاج (Production)

عند نشر الموقع، بدّل إلى:
- **Mailtrap Email API** (للإرسال الحقيقي — نفس المنصة، صندوق منفصل)
- أو خدمة أخرى: SendGrid, Amazon SES, Postmark, Resend

## ❓ استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| `Connection refused` | تحقّق من MAIL_HOST و MAIL_PORT (2525) |
| `535 Authentication failed` | خطأ في USERNAME أو PASSWORD — انسخهما من جديد |
| لا يصل شيء | شغّل `php artisan config:clear` بعد أي تعديل |
| `SSL certificate problem` | أضف `MAIL_ENCRYPTION=null` (كما في الإعداد الافتراضي) |
